# PURATANK — SD 탱크 프라모델 3D 디자인 (v3)

미니 탱크 게임용 전차 4종의 3D 모델링 디자인. 콘셉트는 **"강데포르메 SD + 진짜 프라모델"** —
Meng World War Toons·메탈슬러그풍의 뭉툭한 비율에, 런너(스프루) 2장에서 파츠를 떼어
서로 끼워 조립하는 애니메이션이 가능하도록 실제 킷처럼 파츠를 분할했다.

## 기체 선정 (인기/실루엣 차별화 기준)

| 키 | 기체 | 시대 | 게임 포지션 | 런너 색상 |
|---|---|---|---|---|
| `ft` | 르노 FT | WWI 프랑스 | 소형·기동형 | 샌드 오커 `0xd9a96c` |
| `mk4` | Mark IV | WWI 영국 | 이색·장거리형 | 카키 브라운 `0xa08d5f` |
| `t34` | T-34 | WWII 소련 | 표준·밸런스형 | 올리브 그린 `0x7d9b4e` |
| `tiger` | 티거 I | WWII 독일 | 헤비·보스형 | 저먼 그레이 `0x8d99a6` |

## 데포르메 원칙 (v2 — WWT/메탈슬러그 기준)

- **차체 압축**: 실스케일 대비 길이를 25~30% 더 줄여 정사각형에 가까운 풋프린트.
  (T-34 차체 7.6, 티거 8.2, Mark IV 8.7, FT 5.4 유닛)
- **포탑 거대화**: 포탑 폭이 차체 폭의 60~70%, 길이가 차체 길이의 2/3.
  T-34 달걀 포탑 4.3×4.9, 티거 말굽 포탑 5.2 폭.
- **주포 뚱뚱하게**: 포신 반경 0.36~0.56, 총구로 갈수록 벌어지는 나팔형·대형 머즐브레이크.
- **바퀴 거대·소수화**: 로드휠 3개/측(티거는 겹배열 3+2), 반경 1.0~1.15.
- **궤도 과장**: 클리트(트랙 슈)가 옆·바깥으로 크게 돌출, 차체 높이의 절반을 차지.
- **단일 런너 색상**: 패널라인/음각은 동일 색상의 어두운 셰이드(42% 밝기)로 몰드 그림자 표현.
- **과장 디테일**: 돔 리벳(WWI)·육각 볼트(WWII) 반경 0.10~0.14로 오버사이즈.
- **판 두께**: 하부 차체는 욕조형 쉘, 궤도 피스는 두께 0.42~0.55의 밴드 단면.

## 궤도 = 링크&렝스(link & length) — 전 파츠 평면 몰드

실제 스냅킷 방식 그대로: **직선 "렝스"**(여러 슈가 이어진 밴드)와 **개별 "링크"**가
모두 런너에 평면으로 눕고, 곡선부는 조립 시 링크가 각도를 나눠 바퀴를 다각형으로
감싼다(페이싯이 그대로 보이는 것이 모형미). 각 피스는 한쪽 끝에 수 페그(연결 탭),
안쪽 면에 가이드 스터드를 가진다.

- T-34 / 티거: 측당 렝스 2 + 링크 6~8
- 르노 FT: 측당 렝스 2 + 링크 9 (대형 아이들러 곡선)
- Mark IV: 측당 렝스 4 + 링크 8 (마름모 일주)

`plamo.js`의 `trackLayout(circles, t, linkLen)`이 바퀴 원 배치에서 외접 탄젠트를
계산해 렝스/링크 세그먼트(위치·각도)를 산출하고, `trackLengthPiece`/`trackLinkPiece`가
평면 파츠 메시를 생성한다. 곡선부 안쪽은 로드휠 파츠의 아이들러/스프로킷이 채운다.

## 사출 성형 특성의 재현

- **바닥 뚫린 쉘 포탑**: FT·T-34 포탑은 회전체 쉘(`revolveShell`), 티거 포탑은
  벽 링 압출 + 일체 지붕판 — 런너 위에서 캐비티와 판 두께가 보인다.
- **금형 빼기 방향 준수**: 모든 파츠가 런너 평면에 눕고, 언더컷이 큰 형상
  (곡선 궤도 등)은 분할로 회피.
- **로드휠**: 겹침 없는 개별 디스크를 한 줄로 잇는 평면 파츠 + 눈에 보이는 연결 탭.
  티거의 겹배열은 외측 열/내측 열 2파츠로 분할.
- **셀 구조 런너**: 파츠마다 세로 레일로 구획된 셀 + 짧은 좌/우 게이트 +
  7세그먼트 각인 **파츠 번호 태그**, 런너 기호(A/B) 명판, 중앙 이젝션 링 —
  타미야식 런너 레이아웃을 따랐다.

## 런너(스프루) 사양

- 기체당 **A(대물)/B(궤도·소물) 2장**. 파츠는 행×셀 구조로 자동 배치.
- 프레임/셀 레일 r≈0.3, 게이트 r=0.12 (파츠 좌/우 가장자리 → 인접 레일, 짧게).
- 각 셀에 7세그먼트 각인 파츠 번호 태그, 좌상단에 런너 기호(A/B) 명판, 중앙 이젝션 링.
- 파츠의 `lieRot`(눕힘 회전)으로 런너 평면 정렬 — 트랙 렝스/링크는 실킷처럼 평면으로 눕는다.

## 차고(Garage) 플로우 — 선택 → 개봉 → 조립 → 게임 시작

뷰어 기본 화면은 **책상 위 차고**다: 커팅매트가 깔린 책상 선반에 기체별 박스 4개가
진열되어 있고(각자 자동 생성된 박스아트), 상단 버튼으로 전차를 고르면:

1. **꺼내기(fetch, 1s)**: 선택한 박스가 선반에서 작업 위치로 날아와 커진다.
2. **개봉 + 조립**: 아래 "조립 애니메이션" 시퀀스 진행.
3. **교체**: 조립 중 다른 전차를 고르면 — 만들던 파츠와 런너를 트레이에 고이 담고
   뚜껑을 닫아 원래 선반 자리에 내려놓은 뒤(pack, 2s) 새 박스를 꺼낸다.
4. **▶ 시작**: 게임 핸드오프 — `game/index.html?tank=<key>`가 있으면 그리로 이동,
   없으면 `puratank:start` CustomEvent + parent postMessage + `__pt.onStart` 훅 호출.
   (조립이 끝나지 않아도 시작 가능 — 조립은 연출일 뿐.)

## 조립 애니메이션 — 박스 오픈부터 공구 연출까지

**박스 인트로**(3.4s)로 시작한다: 타미야/아카데미풍 박스아트 킷 박스가 놓여 있고,
뚜껑이 들려 바닥에 놓인 뒤 런너들이 박스에서 나온다. 박스아트는 실행 시점에
완성 모델을 히어로 앵글로 오프스크린 렌더링해 캔버스에 합성한다(하늘 그라데이션,
로고 밴드, 기체명 타이포, SD 배지, 액션 스트릭) — 기체마다 다른 아트가 자동 생성된다.

**런너 스테이징**: 커팅할 파츠가 속한 런너만 작업대(카메라 앞)에 서고, 나머지 런너는
박스에 기대 세워둔다. 커팅 대상 런너가 바뀌면 두 런너가 자리를 교대한다.

파츠는 `order` 순 스태거(0.55s) 파이프라인:

1. **니퍼 커팅**(0.55s): 대형 니퍼(SD 킷 대비 실물 크기감, 원점=날 끝)가 게이트로
   다가와 날을 닫으면 파츠가 분리된다 — **게이트 절단 자국(스텁)**이 남는다.
2. **유리사포 다듬기**(0.55s): 반투명 유리사포가 가장자리를 왕복하며 자국을 없앤다.
3. **비행 + 딸깍 스냅**(0.6s): 베지어 아크로 조립 위치에 착지, 바운스 + 스냅 링.

니퍼/사포는 각 1개라 파이프라인으로 움직이고(사포질 중 니퍼는 다음 파츠 커팅),
전 과정이 시간의 순수 함수라 스크럽 바로 자유롭게 되감긴다.
완성/라인업 뷰의 모델은 절단 자국이 모두 다듬어진 상태다.

## 핀-홀 몰드 (끼움 결합 표현)

- 하부 차체 림 4점 수 페그 ↔ 상부 차체 밑면 암 소켓
- 상부 차체 데크의 포탑 링 소켓(어두운 링), 로드휠 열의 차체 결합 축 페그
- 트랙 피스: +z 끝 수 페그 ↔ −z 끝 암 소켓(어두운 홈), 안쪽 가이드 스터드
- 큐폴라/방순/6파운더 뒷면 페그

## GLTF 익스포트

뷰어의 **⬇ GLTF** 버튼으로 현재 기체의 완성 모델을 `puratank-<key>.glb`(바이너리,
기체당 2.6~3.3MB)로 다운로드한다. 게임 엔진에 바로 임포트 가능.
테스트 API: `window.__pt.exportGlb()` (바이트 수 반환).

## 코드 구조

```
puratank/
  index.html            뷰어 UI (런너/조립 애니메이션/완성/라인업)
  lib/                  three.js r166 vendored
  src/
    plamo.js            프라모델 파츠 빌더 (챔퍼박스, 프로파일 압출, 쉘,
                        링크&렝스 트랙 피스, 패널라인, 리벳/볼트, 로드휠 …)
    runner.js           런너(스프루) 자동 생성기
    tanks/*.js          기체별 파츠 정의 (형상 + 런너 배정/눕힘 + 조립 트랜스폼 + 순서)
    main.js             뷰어 / A·B 런너 / 조립 애니메이션 타임라인
```

게임에서 쓸 때는 `buildXxx()`가 반환하는 파츠 정의를 그대로 가져가면 된다 —
완성 모델은 `assembled` 트랜스폼으로 배치, 조립 연출은 런너 슬롯과 타임라인을 돌린다.

## 리얼타임 카드 전투 (현행)

턴이 없다 — 전장은 계속 흐르고, 플레이어의 입력은 **카드 사용뿐**이다.
기본 5장(전진/후진/좌/우/공격) × 같은 카드 겹치기 = 단계 강화. 방향은
**가장 가까운 적 기준**(`cardBaseAngle`) — 차체가 어디를 보든 전진 = 적에게,
후진 = 적 반대로. 어디로 갈지 항상 예측 가능해야 한다:

- 전진 → 돌격(1.6×) → 전속돌파(1.9×, 기동사격 −12)
- 후진 → 급후진 → 긴급이탈. 후진 1단은 **돌아서지 않는다** — 전 구간 후진
  기어(rev 강제)로 전면장갑을 적에게 문 채 저속 이탈 (정면 장갑 ×0.7 활용).
  2단부터는 돌아서 빠르게 빠지는 것도 허용 — 후면 노출은 대가.
- 좌/우 → 측면전개(제동→선회→새 방향 전진) → 우회(적 측후면 각 +60/+32 가산)
- 공격 → 조준(+15) → 정밀사격(+30, 약점 피해 1.25배) — 연사가 아니라 사격
  품질 강화. 발사는 항상 1회, 이동 중엔 조준 보너스 절반.
- 전진↔후진 = 급제동(상쇄), 다른 이동 = 진로 변경.
- **이동·조준·노출의 상충**: 움직이는 표적은 명중 −12(기동▶사격), 기동사격
  −8(돌격 중 −12), 정지 조준이 최고지만 그만큼 정지 표적이 된다.
- 장갑 방향(전 0.7/측 1.2/후 1.5)과 헐다운(지형이 주는 방어)이 카드 선택의
  이유를 만든다. 상세 규칙은 `/rules` 페이지.

- **핸드/리필**: 시작 2장, 최대 5장. 쓰든 안 쓰든 **3초마다 1장**이 왼쪽
  덱에서 뒤집히며 핸드 왼쪽으로 날아든다 (덱 소진 시 버림 더미 셔플).
  핸드는 부채꼴로 펼쳐진다.
- **카드 사용**: 1탭 = 선택(카드 확대), 같은 카드 재탭 또는 **위로 스와이프**
  = 사용. 사용된 카드는 탱크 머리 위로 날아가 사라지는 연출.
- **수행 시간**: 카드마다 다르다 — 돌격 5s / 후퇴 4.5s / 회피 3.5s /
  공격 2s / 엄호 2.5s / 경계 0.8s(자세). 이동 카드는 수행 시간 동안 기동
  락. **미리 생각해서 내는 것이 중요하다.**
- **차종별 재장전**: reload(턴)×3초 — FT 3s / T-34·MkIV 6s / 티거 9s.
  재장전 중 공격 카드를 내면 **제자리에 멈춰 장전을 기다렸다 쏜다**
  (그동안 기동 불가 — 성급한 공격의 대가). 재장전 중엔 공격 카드가
  어두워진다.
- **카드 합성 = 이전 행동 취소** (실행 중 카드에 얹기): 행동은 언제나
  하나. 얹는 순간 이전 행동이 그 자리에서 취소되고, 합성된 동사가
  **새 행동 하나**로 다시 시작된다 (`routeCard`→`cancelAction2`→`runAction`).
  - 전진 ↔ 후진 (반대 이동) = **급제동** — 취소·정지, 두 카드 상쇄
  - 같은 이동 카드 = **단계 상승** — 한 단계 위 기동(돌격/전개…)을 새로 시작
  - 이동 중 + 다른 이동 = **진로 변경** — 새 방향 1단 기동을 새로 시작
  - 이동 중 + 공격 / 조준 중 + 이동 = **기동사격** — 같은 기동을 새로
    밟으며 달리는 채로 겨눈다 (기동 −8, 조준 보너스 절반)
  - 조준 중 + 공격 = **조준 승급** (즉응→조준→정밀) — 유일한 비취소 합성
- **적 AI**: 개별 두뇌 루프 — 각자 자기 페이스(레벨 높을수록 빠릿)로
  결심→실행 반복. 경계는 자세로 지속되고, 누구든 그 사선을 지나면
  리얼타임 스냅 사격. HUD엔 격파 수(💥 n/m)와 경과 시간(⏱)만.
- 템포는 전체적으로 종전의 절반 (이동 스텝 210ms/칸, 선회 1.3rad/s).

## (구) 전술 카드 덱 — 라운드제 시절 설계

전술은 정해져 있으니 남는 것은 조합과 순서, 그리고 운이다 (슬레이 더 스파이어식).

- **덱 20장**: 돌격×4 · 회피×3 · 후퇴×3 (이동 — **적 기준**: 간격을 1.2칸
  이상 좁히면 돌격, 벌리면 후퇴, 그 사이 측면 기동이 회피) / 공격×4
  (직사·곡사 자동 선택) / 엄호×3 (제압사격) / 경계×3. 재장전 메커닉은
  제거하고 공격 빈도를 카드 희소성이 대신한다 (플레이어 reload 0).
- **이동+공격 동시**: 이동 카드와 공격/엄호 카드를 연달아 내면 **한
  서브턴에 이동하며 사격**한다 (기동 사격 명중 −8, 사격은 이동이 끝난
  새 위치에서 구체화). 경계는 정지 자세라 단독 서브턴.
- **엄호 🔥 = 제압사격**: 표적이 안 보여도 최근접 적의 대략 위치(숨었으면
  크게 흔들림)나 전방 허공에라도 쏜다. 착탄 3칸 내 적은 다음 사격 −15
  (제압, 1회 소모).
- **라운드**: 5장 드로 → 최대 3장을 탭한 순서대로 예약(= 3서브턴).
  **플레이어의 입력은 카드 선택뿐** — 목적지 셀, 사격 표적, 포탑 방향 등
  모든 세부 결정은 실행 시 🎲 주사위와 자동 판단이 내린다. 3D 뷰는
  카메라 조작 전용. 안 쓴 카드는 버리고 매 라운드 새로 5장, 덱이 마르면
  버림 더미를 셔플해 재사용 (시드 기반 — 같은 시드 = 같은 드로).
- **자동 결정 — "똑똑한 조종수" (서브턴마다 그 시점 전장 기준)**:
  돌격 = 접근 + 사격각이 서는 셀 가산 / 회피 = 적 조준선에서 옆으로 크게
  (변위×측면성 가중) / 후퇴 = 전면을 문 채 후진으로 이격. 공통으로
  **경로가 아이템 위를 지나면 크게 가산해 일부러 주워 먹고**(후퇴 중엔
  덜 무리), 헐다운 스팟 보너스, 이동 후 포탑은 최근접 적을 문다.
  공격 = 보이는 적 중 명중률 최고 표적(직사·곡사 불문), 없으면 엄호처럼
  맹목 사격. 항상 최적 수를 둔다 (주사위 운 요소는 제거).
- **명령 하이라이트**: 계획 단계엔 내 전차 발밑에 시안 링이 숨쉬며 돌아
  "지금 이 전차에 명령 중"임을 표시한다.
- **카드 방향**: 투영 원점의 차체 방향 기준 전방 ≤65° = 전진 /
  65~115° = 횡보 / ≥115° = 후퇴(후진 기어).
- **엄호 🛡**: 그 서브턴 받는 피해 ×0.6 + 다음 사격 명중 +10.
- **🎲 2d6 운**: 실행 버튼을 누르면 주사위 2개를 굴린다 — (합−7)만큼
  플레이어 명중 +2/점, 적 명중 −1/점, 합이 10+면 적의 리드 예측이 한 급
  둔해지고 4−면 예리해진다. 라운드 전체에 적용.
- 적 AI는 카드 없이 기존 동사 AI로 응수한다 (매 서브턴 새로 반응).

## 게임 규칙 — 동시 턴제(WeGo) · 3동사

- 한 턴 = **동사 하나** (기동/사격/경계), 입력하는 순간 전원 동시 해결.
  상성 삼각형: **기동 ▶ 사격**(예측탄 회피) · **경계 ▶ 기동**(스냅 사격) ·
  **사격 ▶ 경계**(정지 표적).
- **다중 턴 예약 (최대 2턴)**: 한 번에 최대 2개의 동사를 미리 예약한다.
  기동을 예약하면 **도착 자세 고스트가 남고, 그 지점에서 이어서 다음
  동사를 계획**한다(투영 원점 이동) — 예약 고스트가 순번 뱃지 + 연결선으로
  체인처럼 보인다. 필드/사거리·재장전 판정도 투영 원점 기준으로 계산.
  **사격은 고스트에서 나간다** — 예약된 이동이 있으면 사격 드래그를
  마지막 고스트에서 시작하고, 탄도 프리뷰·포탑 조준·실제 발사가 전부
  그 고스트 지점 기준이다(예약 없으면 현재 전차). 2턴이 차거나
  **▶ 실행** 버튼을 누르면 예약을 순차 서브턴으로 실행하고, 적은 매
  서브턴 새로 반응한다. ESC로 마지막 예약 취소.
- **해결 순서**: ① 전 차량 동시 이동 — 경계 스냅은 이동 도중 **실시간**으로
  발동 ② 이동이 모두 끝난 뒤 일반 사격 발사. 목적지를 맞게 예측한 셀
  사격은 도착한 적을 정통으로 맞춘다.
- **기동**: 이동 필드의 셀을 터치하면 도착 자세 고스트가 보이고,
  **드래그로 "도착 후 포탑 방향"을 지정**한다(고스트 포탑이 따라 돎) —
  놓으면 실행되고 도착 후 포탑만 그 방향으로 선회한다(차체는 경로 방향).
  **8방향(대각 포함) 이동**에 45° 단위 회전 비용(0.7)이 크게 붙어,
  현재 차체가 향한 쪽으로는 **훨씬 멀리 간다** (비대칭 이동 필드). 차체 선회는
  각속도 기반으로 느리고(주행 전환 2.4rad/s), 이동이 끝나면 마지막 진행
  방향 그대로 — 추가 제자리 선회는 없다. 지나간 자리에는 좌우 궤도 리본
  데칼이 부드럽게 이어져 남는다 (링버퍼 1100세그먼트, 드로콜 1).
- **이동 비용 모델**: 전진·후진 스텝 = 1(평지 기준, 후진도 동일 비용),
  회전은 90°당 +1.5(=45°당 0.75)가 더해져 이동 반경이 정해진다 —
  차체 축(전·후진) 방향으로는 멀리, 옆으로 틀려면 회전값을 크게 치른다.
- **구간별 전/후진 기어**: 경로 탐색이 (셀, 차체 방향) 상태 공간에서
  스텝마다 전진(차체=진행 방향) / 후진(차체는 반대 유지)을 선택한다 —
  등 뒤 목적지는 차체를 돌리지 않고 후진으로 빠져 **이동 중에도 목표
  각도(전면 장갑 방향)를 최대한 유지**한다. 이동 필드는 같은 파랑 안에서
  **회전 없이 전/후진만으로 닿는 각도 유지 구간을 더 진하게** 음영 처리 —
  선회가 필요한 바깥 구간은 옅은 파랑, 별도 반경처럼 도드라지지 않는다.
- **포탑은 미리 돌지 않는다**: 포탑이 도는 순간은 둘뿐 — 사격 시작 시의
  조준 선회(aimAt), 이동 도착 시 드래그로 지정한 turretYaw 정렬.
  해결 페이즈 중 적 포탑이 플레이어를 계속 물고 도는 사전 추적은 제거
  (경계 사선 읽기가 성립하도록).
- **사격 = 드래그**: 내 차량을 잡고 **목표까지 드래그**하면 탄도가 굵은
  발광 튜브로 표시된다(셰브론 화살표가 탄착점 방향으로 흐르는 애니메이션
  + 글로우 + 레티클). 탄도는 **직사**(빨강, 거의 직선) vs **곡사**(주황,
  높은 포물선 — 차폐·엄폐·헐다운 무시, 명중 −12%·산포 ×1.6·최소 4칸)가
  자동 전환되고, 사격 필드도 빨강/주황 2톤으로 갈라져 보인다 —
  평상시에도 옅은 채움으로 항상 읽힌다.
- **드래그 취소**: 사격/이동 드래그 모두 내 전차 위로 되돌려 놓으면 취소,
  ESC로도 취소. 사격은 유효 셀 밖에서 놓아도 취소된다.
- **시대별 실포탄**: 차체별 실제 사용 탄을 SD 스케일로 재현 — FT 37mm
  Puteaux(짧고 뭉툭) · MkIV 6파운더(둥근 코) · T-34 76mm BR-350(검은 탄두) ·
  티거 88mm PzGr.39(흰 탄도캡), 전 탄종 구리 회전탄대. 탄체는 비행 방향으로
  정렬되어 강선 회전하며, 주황 예광 트레일(30점 라인)이 궤적을 그린다.
  비행 속도는 잘 보이게 느리게 (직사 ~1.7s, 곡사 ~2.6s).
- **탄도 충돌**: 직사는 부앙각 한계로 클램프한 실제 탄도를 레이로 추적 —
  능선·수목·건물·다리·차량(아군 포함) 무엇이든 **걸리면 그 지점에서
  폭발**한다. 조준 셀에 탄도가 닿지 않으면 직사 불가로 표시되고
  곡사로 넘어간다. 유닛 조준점은 포탑 높이(+1.4)라 근거리 직사도 성립.
- **경계**: 상단 👁 칩 — **이동·사격과 같은 1턴 액션으로 큐에 쌓인다**
  (두 턴 연속 경계 가능, 재장전 중엔 ⏸ 대기로 대체). **포탑은 선회하지
  않고 지금 바라보는 방향을 지킨다** — 사거리 안 + 현재 포 방향 ±12°
  이내로 적이 들어온 순간에만 스냅 사격(명중 −8, 경계자당 1회). 이동
  드래그로 미리 지정해 둔 포탑 방향이 곧 경계 사선이 된다. 아무도 안
  걸리면 **조준 스택 +15%**가 쌓여 다음 사격에 가산된다(발밑 금색 링).
- **재장전**: 발사하면 기체별 재장전 턴(FT 1 / MkIV·T-34 2 / 티거 3).
  장전 중엔 사격·경계가 잠겨 기동만 가능 — 차종별 리듬이 갈린다.
- **예측 사격**: 포격은 유닛이 아니라 "셀"을 조준한다. 목표가 그 턴에
  이동하면 빗나가므로 상대의 다음 위치를 예측해서 쏴야 하고, 반대로
  내가 움직이면 적탄을 흘릴 수 있다 (이동 ↔ 사격의 트레이드오프).
- **충돌**: 두 차량이 같은 칸에 진입하려 하거나(맞진입·자리 맞바꿈·
  멈춘 차량 추돌) — 서로 HP가 깎이고 직전 타일에서 멈춘다.
  스텝 단위 동시 시뮬레이션으로 판정.
- **차종별 포 특성**: 부앙각 한계와 고정포 여부가 기체마다 달라 전략이 갈린다.
  FT −18°~+30°(유연) · Mark IV −12°~+8° **좌우 스폰슨 부포**(측면 90°±55°
  사각만 조준 가능 — 사격 필드가 나비꼴, 이동으로 측면을 내줘야 함) ·
  T-34 −5°~+25°(내림각 나쁨 — 언덕 위에서 근접 아래를 못 쏨) ·
  티거 −8°~+15°(장거리 평탄 탄도).
- **Mark IV 트윈 스폰슨**: 좌/우 스폰슨에 6파운더가 **각각** 달려 있고
  각 포는 자기 마운트에서 독립 선회한다 — 목표가 있는 쪽 부포만 사각 안에서
  조준·발사하고(그 포의 포구에서 탄이 나감), 반대쪽 부포는 자기 측면(±90°)을
  겨눈 채 대기한다. 경계 사격도 좌/우 어느 포든 정렬되면 발동. 실차의
  양측 스폰슨포를 그대로 옮긴 조준·연출.
- **가변 맵**: 가로·세로 각각 60~120칸이 시드로 독립 결정(비정사각 가능),
  1타일 = 1유닛. 면적 배율(AREA_F)에 비례해 적 수(2~6)·식생 수가 늘고
  카메라/포그/섀도 범위(MSCALE)도 함께 스케일. 강은 82% 확률로 존재하고
  폭·굽이가 구간별로 변한다(없는 마른 맵도 나온다). 유닛끼리 체비쇼프
  1칸 이내 접근 불가, 동시 이동 충돌도 근접(1칸) 기준.
- **AI 레벨(적 HP바의 AI 수치)**: Lv1은 플레이어의 현재 칸만 조준,
  Lv2는 지난 턴 이동 방향의 절반을, Lv3은 그대로 외삽한 칸을 리드 사격.
  Lv2+는 사선이 능선에 막히면 **그 지형을 포격해 크레이터로 포각을 확보**(굴착).
- **AI 장갑 방향 관리**: 적은 도착 자세가 플레이어에게 측면/후면을
  내주는 이동을 감점(측 −10/레벨, 후 −22/레벨)해 **전면을 유지하며
  기동**한다 — Lv1은 무신경, Lv3은 예민(실측: Lv3은 상대각 5~18°,
  Lv2는 25~54° 유지). 지금 자세가 이미 노출이면 제자리보다 이동을 택한다.
- **거리 명중률**: 4칸 이후 칸당 −3% + 원거리 가속 페널티(1칸=1유닛),
  빗나감 산포 반경도 거리에 비례해 커진다.
- **자동 조준**: 포탑 기체(FT/T-34/티거)는 포탑 요 회전, Mark IV는 고정포가
  사각 안에서 좌우 미세 조준, 그 외엔 차체 선회로
  목표를 향한다. 포신 부앙각은 고도차에 맞춰 자동 조절, 커서 호버 시
  포탑/포신이 미리 따라 움직이는 조준 프리뷰.
- **범위 필드**: 이동/포격 영역 모두 셀 경계 루프를 차이킨 3회 세분해
  타일 느낌 없는 부드러운 곡선 필드로 그린다 (내부 채움은 0.45 간격
  지형 밀착 래스터, 외곽선은 캣멀롬 튜브).
  직사 필드(빨강)는 부앙각·사거리·사선이 전부 유효한 셀만, 곡사 필드(주황)는
  직사 불가지만 포물선으로 넘겨 쏠 수 있는 셀(4칸~사거리)을 표시한다.
  적 AI(Lv2+)도 사선이 막히면 곡사로 넘겨 쏜다 — 언덕 뒤가 안전하지 않다.
- **겹침 해법**: 평소엔 이동 필드를 채우고 사격 필드는 얇은 외곽선만 —
  내 차량에서 드래그를 시작하면 사격 필드가 채워지며 강조된다.
  제스처 시작 위치(차량 위 = 사격, 필드 셀 = 기동)로 의도가 구분된다.
- **HUD**: 상단 중앙 = 턴 + 날씨·시간대 + **계획 breadcrumb**
  `[① 이동 ▸ ② 사격]`(예약 확정/미리보기(초록 ?)/빈 슬롯 구분) +
  경계·▶실행·✕취소 칩 + 힌트 한 줄. 좌상단 = 플레이어 섬네일, 우상단 =
  적 섬네일 세로 스택(3D 초상화 + Lv + HP바, 탭하면 상세 패널 + 카메라가
  그 전차로 이동). HP는 전차 위 바에 숫자 병기. 좌하단 = FPS 위젯
  (클릭 시 프레임타임·드로콜·트라이 펼침), 우하단 = ⚙ 설정 다이얼로그
  (다시 시작 / 게임 종료→차고 / 날씨·시드 표시). 하단 메뉴는 없다 —
  액션 확정용 ✓ 바만 하단 중앙에 뜬다.
- **모바일 탭 2단계 조작**: 드래그(데스크톱)와 별개로, 셀 "탭" = 고스트
  미리보기(+고스트 근처 드래그로 포탑 방향 조정) → ✓ 확정. 내 전차 탭 =
  🎯 사격 모드 → 목표 탭 = 탄도 미리보기 → ✓ 발사(버튼에 명중률 표시).
  ESC/✕가 미리보기 → 제스처 → 마지막 예약 순으로 취소.

## 전차전 레이어 — 다리 / 헐다운 / 정지 사격 / 장갑 방향

- **다리**: 깊은 강 구간(여울에서 가장 먼 곳)에 목교가 놓인다 — 안전한
  횡단로지만 내구도 100, 포격 2~3발이면 붕괴. 위에 있던 차량은 강으로
  추락(피해 40)해 기어 나와야 한다. 다리 위 통행 비용 1.2.
- **헐다운 스팟**: 능선 마루 칸을 자동 감지해 금색 링으로 표시 (최대 10곳,
  서로 4칸 이상 이격). 그 위의 목표는 차체가 가려져 피격 확률 −16%.
- **정지 사격 보너스 (halt fire)**: 직전 턴 정지 시 명중 +12%, 기동 사격
  −8%. 정지 + 장전 완료면 발밑에 금색 조준 안정 링이 맥동하고,
  명중률 표기에 ★가 붙는다.
- **장갑 방향**: 피격 방향에 따라 전면 ×0.7 / 측면 ×1.2 / 후면 ×1.5.
  피격 시 "전면 장갑!/측면 관통!/후면 직격!" 텍스트 + 사수 방향 화살촉
  플래시로 어디서 맞았는지 화면에 표시. 차체 방향 지정(고스트 드래그)이
  방어에도 직결된다.
- **조우 구성**: 1(플레이어) : 2(적) — 적은 모두 같은 차체(시드 선택).

## 시간대·날씨 & 시야

지형 rng와 분리된 시드 분기로 매 판 **시간대(새벽/낮/석양/밤) × 날씨(맑음/
흐림/안개/비/폭우)**가 정해진다 (`?tod=` `?wx=`로 고정 가능, 같은 시드 =
같은 지형 유지).

- **조명 프리셋**: 시간대별 태양 위치·색·강도, 반구광, 포그 색, 톤매핑
  노출, 컬러 그레이드(채도/웜틴트)가 통째로 바뀐다 — 석양은 낮은 주황
  롱섀도, 밤은 푸른 달빛. 하늘 캔버스도 프리셋: 맑은 밤엔 별 340개 + 달
  원반, 흐림 계열엔 먹구름 띠(폭우일수록 검게), 안개는 잿빛 균일 덮개.
- **비/폭우**: 카메라 타깃을 따라다니는 빗줄기 LineSegments(비 1300 /
  폭우 3000, 기울어진 낙하 + 표류), 루프 빗소리(밴드패스 노이즈), 폭우엔
  **번개**(하늘 전체 2~3회 깜빡 + 지연된 천둥 럼블). 식생 바람도 날씨별
  배율(셰이더 uWindA: 폭우 2.4배 + 풍속 1.9배)로 거세진다.
- **시야 제한 (안개 9칸 / 폭우 13칸)**: 한계 밖의 적은 **렌더링 자체가
  꺼진다** — 조준 레이캐스트·타깃 링·경계 스냅에서도 제외되고, 섬네일은
  블러 + `?`로 바뀌며 탭해도 위치를 알려주지 않는다. 대신 시야 밖 전차가
  움직이면 **풀 튕김이 증폭되고 지나는 길의 나무·수풀이 술렁**여 낌새만
  남는다 (+궤도 자국·엔진음). 적 AI도 같은 시야를 쓴다: 시야 밖
  플레이어에겐 사격 계획을 세우지 못하고 소리로 접근·매복(경계)만 한다.
  지형 조준 포격(맹목 사격)은 양쪽 다 가능.

## 차체 자세 — 서스펜션 & 격파 분해

- **4점 접지 자세**: 좌우 궤도 앞뒤 4점의 지형 높이로 pitch/roll 목표를
  구한다. rotation.order='YXZ' 기준 전방 y성분=−sin(pitch), 우측
  y성분=+sin(roll) — roll 부호가 반대였던 버그를 수정 (옆 경사에서 비탈
  안쪽으로 눕던 문제).
- **로드휠 서스펜션**: 자세를 즉시 스냅하지 않고 감쇠 스프링(K42/D9.5,
  약한 언더댐핑)으로 따라간다 — 둔덕을 넘을 때 차체가 출렁이고, 크레이터로
  발밑이 꺼지면 부드럽게 내려앉는다 (궤도 유격의 여유감).
- **궤도 링크별 처리 (킷 어댑터 개편)**: 트랙 피스(렝스/링크)를 차체 병합에서
  분리해 좌/우 트랙 밴드로 따로 병합(+2드로/기체)하고, 병합 전에 피스별
  서브메시(슈·페그·스터드)와 탱크 로컬 행렬을 레지스트리에 기록한다.
  - **처짐(sag)**: 조립 시 상부 런 피스를 현수선 가중치(sin π t)로 내리고
    낱장을 미세하게 들썩여 베이크 — 개체마다 유격량이 달라(0.18~0.34 킷
    유닛) 런타임 비용 없이 궤도가 늘어진 느낌이 난다. 하부 런(접지면)은
    그대로.
- **격파 시 하부 분해 (60%)**: 병합된 트랙 밴드를 끄고 레지스트리의
  **실제 킷 렝스/링크 16~20피스**가 낱개로 좌우에 흩어진다 +
  로드휠 2~3개가 옆으로 굴러 나간다.
- **AI 편대 간격**: 적끼리 2.5칸 미만 도착지는 사실상 금지(−90),
  4.5칸 −45, 7칸 −14 감점 — 뭉쳐서 파편·스플래시 연쇄로 자멸하지 않고
  간격을 벌린 편대로 움직인다. 먼저 계획한 아군의 도착 예정 칸도 피한다.
- **경사지 건물 기초**: 바닥 아래 빈 공간을 다진 흙 기단(최저 지면보다
  깊이 박히는 2단 프러스텀) + 井자로 어긋 쌓은 침목 크립이 메꾼다.

## 파편 물리 — 프레임레이트 독립

격파 부품·건물 데브리·풀 튕김의 물리는 트윈 진행률(k) 증가분 × 총 시간으로
dt를 산출한다(`kDt`). 고정 0.016/프레임 방식은 저FPS 모바일에서 시뮬 시간이
트윈 길이보다 짧아져 **파편이 공중에 얼어붙는** 버그가 있었다. 격파 부품
트윈이 끝나면 아직 공중인 조각을 지면에 내려놓는 안전망도 있다.
